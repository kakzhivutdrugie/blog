ИНСТРУКЦИЯ ПО РАЗМЕЩЕНИЮ БЛОГА НА GITHUB PAGES
═══════════════════════════════════════════════════════════════════════

⏱️ Время настройки: 15-20 минут
💰 Стоимость: БЕСПЛАТНО навсегда
🚀 Результат: blog.kakzhivutdrugie.ru работает автоматически

═══════════════════════════════════════════════════════════════════════
ШАГ 1: РЕГИСТРАЦИЯ НА GITHUB (5 минут)
═══════════════════════════════════════════════════════════════════════

1. Открой сайт: https://github.com
2. Нажми "Sign up" (Зарегистрироваться)
3. Введи данные:
   - Email: твоя почта
   - Password: придумай надёжный пароль
   - Username: например, kakzhivutdrugie (будет в адресе)
4. Подтверди email (придёт письмо)

✅ Готово! Теперь у тебя есть аккаунт.

═══════════════════════════════════════════════════════════════════════
ШАГ 2: СОЗДАНИЕ РЕПОЗИТОРИЯ (2 минуты)
═══════════════════════════════════════════════════════════════════════

1. Войди в GitHub
2. Нажми зелёную кнопку "New" (или "Create repository")
3. Заполни:
   - Repository name: blog
   - Description: Блог "Как живут другие"
   - Public (оставь как есть)
   - ✓ Initialize with README (поставь галочку)
4. Нажми "Create repository"

✅ Репозиторий создан!

═══════════════════════════════════════════════════════════════════════
ШАГ 3: ЗАГРУЗКА ФАЙЛОВ (5 минут)
═══════════════════════════════════════════════════════════════════════

СПОСОБ А: Через веб-интерфейс (проще)
----------------------------------------
1. Открой свой репозиторий на GitHub
2. Нажми "Add file" → "Upload files"
3. Перетащи ВСЕ папки и файлы из blog-system:
   - index.html
   - sitemap.xml
   - папка articles/ (со всеми статьями внутри)
4. Внизу страницы:
   - Commit message: "Первая версия блога"
   - Нажми "Commit changes"

СПОСОБ Б: Через GitHub Desktop (для продвинутых)
-------------------------------------------------
1. Скачай GitHub Desktop: https://desktop.github.com
2. Установи и войди в аккаунт
3. File → Clone repository → выбери свой blog
4. Скопируй файлы из blog-system в папку репозитория
5. Commit → Push

✅ Файлы загружены!

═══════════════════════════════════════════════════════════════════════
ШАГ 4: ВКЛЮЧЕНИЕ GITHUB PAGES (3 минуты)
═══════════════════════════════════════════════════════════════════════

1. В репозитории нажми "Settings" (⚙️ справа вверху)
2. Слева найди "Pages"
3. В разделе "Source":
   - Branch: выбери "main" (или "master")
   - Folder: оставь "/ (root)"
4. Нажми "Save"
5. Подожди 1-2 минуты

✅ GitHub покажет: "Your site is published at https://USERNAME.github.io/blog/"

Проверь: открой этот адрес — должен открыться твой блог!

═══════════════════════════════════════════════════════════════════════
ШАГ 5: ПОДКЛЮЧЕНИЕ ДОМЕНА blog.kakzhivutdrugie.ru (5 минут)
═══════════════════════════════════════════════════════════════════════

ЧАСТЬ 1: Настройка на GitHub
-----------------------------
1. В Settings → Pages найди "Custom domain"
2. Введи: blog.kakzhivutdrugie.ru
3. Нажми "Save"
4. ✓ Поставь галочку "Enforce HTTPS" (когда появится)

ЧАСТЬ 2: Настройка DNS (где купили домен)
------------------------------------------
Зайди в панель управления доменом (где покупала kakzhivutdrugie.ru)

Добавь CNAME запись:
┌────────────┬──────┬────────────────────────────────────┐
│ Имя        │ Тип  │ Значение                           │
├────────────┼──────┼────────────────────────────────────┤
│ blog       │ CNAME│ USERNAME.github.io                 │
└────────────┴──────┴────────────────────────────────────┘

Где USERNAME — твой логин на GitHub

Пример:
- Если твой GitHub: github.com/kakzhivutdrugie
- То значение: kakzhivutdrugie.github.io

⏳ Подожди 5-30 минут (DNS обновляется)

✅ Готово! Теперь blog.kakzhivutdrugie.ru открывает твой блог!

═══════════════════════════════════════════════════════════════════════
ШАГ 6: ДОБАВЛЕНИЕ ССЫЛКИ НА ТИЛЬДЕ (2 минуты)
═══════════════════════════════════════════════════════════════════════

1. Зайди в редактор Тильды на kakzhivutdrugie.ru
2. Добавь блок "Кнопка" (T113, T123 или любой с кнопкой)
3. Настрой кнопку:
   - Текст: "Блог" или "Статьи"
   - Ссылка: https://blog.kakzhivutdrugie.ru
4. Опубликуй изменения

✅ Теперь с главной страницы можно перейти в блог!

ЧАСТЫЕ ВОПРОСЫ
═══════════════════════════════════════════════════════════════════════

✅ БЕСПЛАТНО навсегда. GitHub Pages бесплатный для публичных сайтов.

❓ Как добавить новую статью?
✅ Просто загрузи новую папку в articles/ через GitHub (как в Шаге 3).

✅ GitHub Pages очень быстрый! Если медленно — проблема в интернете.
